# Rules engine documentation index

This index exists because the severity-scoring documentation grew large
enough that architecture and calibration questions were being discussed
in the same breath, even though they answer different questions and
carry different levels of certainty. Part A is settled, implemented, and
tested. Part B is calibration analysis on top of Part A — real findings,
but corpus-level, single-reviewer, and in several places preliminary.
Read them as separate conversations.

## Part A — Deterministic rule engine (detection, not severity)

How a clause gets found and matched in the first place. This layer is
implemented, tested (261+ tests), and not part of any open calibration
question.

- `rules_engine_overview.md` — how the engine works end to end: chunking,
  pattern/anchor matching, `RuleClass` (presence-risk vs required-section)
- `rule_structure.md` — the `Rule` dataclass fields used for detection
  (pattern, anchors, nearby, window, topic/protective patterns)
- `rule_categories.md`, `all_rules.md`, `example_rules.md` — the rule
  catalog itself, organized by severity tier and practice area
- `versioning_strategy.md` — how `rules/version.json` and rule-count
  changelogs are maintained

## Part B — Severity calibration (WAS, ceilings, thresholds)

Given a matched clause, how severity gets assigned. This is where the
open questions live — the scoring *engine* (`severity_scoring.py`) is
implemented and tested, but the *calibration* of its thresholds against
real rules is active, corpus-level analysis, not a finished spec.

**Frozen architecture (the rubric itself, approved and not to be
redesigned without governance sign-off):**
- `severity_architecture.md` — the v1.0 spec: 11 factors, 8 ceiling
  rules, WAS formula, thresholds, metadata schema, jurisdiction model
- `severity_implementation.md` — how that spec was turned into code:
  engine, validators, migration tooling, contributor workflow

**Migration (applying the frozen spec to the 117 real rules):**
- `severity_migration_report_full_117.md` — every rule's legacy vs.
  computed severity, generated by `scripts/generate_full_migration_report.py`
  from `scripts/severity_migration_full.py`. This is a single-reviewer
  pass ("Reviewer 1 (Framework Author) Migration") — the blind re-score
  gate and attorney sign-off have not happened yet.

**Calibration experiments (analysis run against the migration data, in
the order they were produced — each one is a response to what the
previous one found, not an independent study):**
1. `severity_practical_max_experiment.md` — per-rule ceilings given each
   rule's own already-scored factors
2. `severity_family_clustering_experiment.md` — same question, grouped
   by practice-area family; the "no single threshold fits every family"
   observation
3. `severity_family_conclusion_stress_test.md` — an adversarial attempt
   to disprove #2 by testing five alternative explanations
4. `severity_authoring_bias_and_confidence.md` — whether the family
   pattern is actually an authoring artifact, plus a per-family
   confidence table by sample size
5. `severity_threshold_v1_1_candidate.md` — a tested, implemented fix
   proposal responding to #2's finding (`compute_severity(mode="relative")`
   in `severity_scoring.py`), **not yet the default** and not yet
   governance-approved. Cuts corpus-wide LOW from 94→49 rules and brings
   MEDIUM from 0→40, but two families (Administrative, Insurance) remain
   fully LOW for a specific, already-diagnosed reason — see the doc.

### How much weight to put on Part B's findings right now

- The scoring *engine* is solid: 361 tests, deterministic, matches its
  own spec exactly (both the frozen absolute mode and the relative
  candidate mode).
- The calibration *findings* are real but should be read at the
  confidence level stated in `severity_authoring_bias_and_confidence.md`
  — 8 of 18 families have n≤4 rules and are explicitly preliminary. The
  core cross-family observation is corroborated by several
  High/Medium-confidence families independently, but any claim about one
  specific small family's exact numbers is a 2-4 rule observation, not a
  settled property.
- None of Part B has been through the §10 blind re-score gate (a second,
  independent reviewer). Everything here is Reviewer 1's single pass.
  Treat every calibration conclusion as a hypothesis for that review, not
  a final answer — this is stated explicitly in each document, not just
  here.
- No thresholds, weights, or ceiling rules have been changed as a result
  of any Part B finding. All four calibration documents are analysis
  only, per explicit instruction to understand before modifying.
